# G3 引用格式說明 (SPACE_CORE v3.02)

## 規則變更
- **格式**：正文句尾使用 `^[來源簡稱]`（例：`^[民法§184]`、`^[判決字號]`、`^[契約書.pdf]`）。
- **目的**：取代舊有 `[1][2]…` 或 `（舊式S1）` 等形式，提供更簡潔且易於辨識的引用標記，同時保持可追溯性。

## 具體應用

### 1. 段落內引用
- 第一次引用時，於句尾加上 `^[來源簡稱]`。
- 重複引用同一來源時，可省略標記。

### 2. 段落末尾來源列表
- 當段落包含一個或多個引用時，段落末尾必須附加：
  ```markdown
  【本段資料來源】
  ^[來源簡稱1]
  ^[來源簡稱2]
  ```

### 3. 全文末尾來源彙總
- 若全文引用的來源超過三個，則在文末提供完整列表，包含詳細資訊（如標題、URL、來源類型、信度分級等）。

### 4. 應避免
- **舊式標記**：絕對禁止使用 `[1][2]…` 或 `（舊式S1）` 等形式。
- **過度標記**：除非必要，否則避免在每個句子後都加上引用，以保持閱讀流暢性。

## 版本演進
- **v1.0**：密集 `（舊式S1）（舊式S2）` 格式。
- **v2.0**：優化為 `[1][2]` 格式 + 末尾統一表，減少干擾。
- **v2.1**：引入 `[T1][T2]` RAG 引用。
- **v2.2**：新增 C5.4 來源可信度分級對應，銜接核心閘門。
- **v2.3**：引用格式統一為句尾上標 `^[來源簡稱]`，並更新所有相關範例與驗證規則。
- **v3.01 (SPACE_CORE)**：G3 規則強調使用 `^[來源簡稱]`，禁止舊式 `[1][2]`。
- **v3.02 (SPACE_CORE)**：進一步確認 G3 格式為 `^[來源簡稱]`，並更新文件標頭與版本。
