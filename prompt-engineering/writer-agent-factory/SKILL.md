# 寫手代理工廠

將寫作主題轉為可直接使用的「寫手系統指令」：識別領域 → 只問缺漏參數 → 生成指令 → 閘門檢核。源自動態多領域寫手生成系統 v4.1。不包含通用提示詞優化（該職責屬 `prompt-factory-7-1`），本技能交付的是寫手指令，非文章正文。

鐵則：**品質優先於 token 效率**；**結構服務內容**；**過度結構化本身即是 AI 味來源**。

## When to Use

- "幫我生成一個寫手"
- "為這個主題建立寫作 agent"
- "產生寫手系統指令"
- "把這個題目交給寫手處理"
- "修改上次的寫手指令"（觸發 S4 修訂）

## Prerequisites

- 無外部金鑰需求。
- 端到端模式需可發起獨立模型呼叫（Council 或任一第二模型）作為 Judge。
- `memory` 工具用於記錄寫手指令版本歷史（S4 修訂鏈）。

## How to Run

1. 透過 `terminal` 或對話解析輸入 → 狀態判定（S0-S4）
2. 缺參數 → 只列缺漏問題並停止；參數足 → 生成寫手指令
3. Gate A 指令自檢 → 修復（最多 2 輪）
4. 交付指令，以 `memory` 記錄版本

## Quick Reference

**狀態機**
- S0_IDLE：非寫作主題 → 「請提供您的寫作主題，我將提出關鍵問題後生成寫手系統指令。」
- S1_ASKING：有主題、缺參數、未要求略過 → 只列缺漏問題，不生成
- S2_GENERATING：參數足或要求略過 → 已知參數＋預設值生成
- S3_DONE：已交付 → 等待新主題或修訂
- S4_REVISE：修改要求 → 版本遞增 v\<N+1\>，只改指定部分

**參數預設值**：受眾=中階從業者 | 重點=原理講解 | 範例=由寫手判斷 | 風格=理性/深度展開/中密度

**溫度建議**（僅 API 場景）：法律/技術 0.2-0.3 | 科普 0.4-0.6 | 敘事 0.6-0.8 | 批判 0.5-0.7

## Procedure

### P1 — 狀態判定

依 Quick Reference 狀態機分流。S4 修訂時：舊版先存 `memory`，只修改指定部分，修改處加側邊註記。

### P2 — 領域識別與提問（S1）

領域：技術 | 心理學 | 法律 | 科普 | 商業 | 其他，可多選。跨領域落差大時合併提問但保留各領域專用選項。只列缺漏項；未答項依預設值，全數列入「假設說明」。

| 問題 | 選項 |
|------|------|
| Q1 受眾 | 一般大眾/入門 | 中階從業者 | 高階專家 | 其他 |
| Q2 重點* | 原理講解 | 實務步驟 | 決策框架 | 常見錯誤 | 其他 |
| Q3 範例* | 詳細 | 關鍵 | 不需 | 由寫手判斷 |
| Q4 風格 | 語氣（理性/敘事/批判/教學）＋節奏（精簡/標準/深度展開）＋密度（低/中/高） |

*選項可依領域替換。

### P3 — 生成寫手指令（S2）

人性化規範（只給原則，**嚴禁具體引導句式範例**）：句長自然交替、避免機械序列詞、以場景或提問開場但不規定句式、適度主觀、段落留白、結尾可呼應開頭但不強制。

模型感知為**條件觸發**：僅當使用者指明目標模型時注入（DeepSeek=thinking 多層推理 | Gemini=先大綱後展開 | Claude=段落逐步展開），未指明則省略以維持模型無關。

**輸出模板**：

```
# 寫手角色：〔依主題命名〕

## 角色定位
〔1-2 句，標註受眾、領域、風格三參數〕

## 假設說明
〔條列所有推測項與依據〕

## 核心任務
依使用者主題，產出符合本指令所有規範的完整文章。

## 輸出要求
自適應 Obsidian 排版；依受眾調整深度；人性化寫作（不套模板句式）；
合格 Frontmatter + 假設說明；套用風格三參數；台灣繁體中文；字數不限。

## 引用格式（非顯而易見主張三擇一，嚴禁未標註的虛構引用）
[來源：作者/機構, 年份] | [來源：URL] | （基於現有知識推論）

## 交稿前自檢（W 閘門，全數通過方可輸出）
W1 引用完整 | W2 無單段>全文40% | W3 無相鄰重複段 |
W4 機械序列詞≤1/千字 | W5 術語密度符合受眾

## 限制
不編造引用或數據；不偏離主題；Frontmatter 除 {{date}} 外不得用占位符
```

### P4 — Gate A：指令層檢核（二元，本技能內執行）

| 閘門 | 通過條件 |
|------|---------|
| A1 段落完整 | 模板七段落齊備 |
| A2 假設說明 | 所有推測參數已條列 |
| A3 無句式範例 | 全文無具體引導句式 |
| A4 模型提示合規 | 未指明目標模型時無模型專屬提示 |
| A5 引用格式嵌入 | 三擇一格式與 W 閘門均在指令內 |

未通過項自動修復，最多 2 輪；仍未過 → 標註 ⚠️ 交付並說明。

### P5 — 交付與版本記錄

交付寫手指令，`memory` 記錄：技能名、版本號、主題、參數組合、時間。

### P6 —（選用）端到端模式

Hermes 直接執行寫手產文時，W1-W5 改由**獨立 Judge**（第二模型）評分 1-5，不得由寫手自評。門檻：W1=5 為硬條件，W2-W5 ≥4；FAIL 回饋修正最多 2 輪，W1<2 一票否決 STOP。

FEG-L3 映射（語法依共享參考 `prompt-factory-7-1/references/feg-core-dsl.md`）：
```
FEG_CORE_EXTREME[
D{CIT,STRUCT,DUP,AIF,AUD};S1..5;
P:CIT>=5&STRUCT>=4&DUP>=4&AIF>=4&AUD>=4;
R:CIT<5|STRUCT<4|DUP<4|AIF<4|AUD<4;
Dg:CIT<3;C:AUD<3;B:CIT<2|SF|QF;
M:P>DLV;R>RTY(max=2);Dg>SAFE;C>ASK;B>STOP
]
```
維度映射：CIT=引用 / STRUCT=結構 / DUP=重複 / AIF=AI味 / AUD=受眾

## Pitfalls

- **Gate A 查指令、Gate W 查文章**，執行主體不同，勿混用（v4.0 曾發生閘門錯位）。
- **自評分數會膨脹**：端到端模式的 W 檢核必須由獨立模型執行，不可讓寫手自評。
- **嚴禁在寫手指令中加入具體引導句式範例**——這正是 AI 味的來源。
- **模型感知僅條件觸發**：未指明目標模型時注入模型提示即違反 A4。
- **FEG 壓縮版僅供快速參照**，不可作為寫手指令本體交付。
- **S4 修訂採最小變更**：只改指定部分，禁止藉修訂重寫全文。
