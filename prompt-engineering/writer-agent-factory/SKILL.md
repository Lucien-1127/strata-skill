# 寫手代理工廠

將寫作主題轉為可直接使用的「寫手系統指令」：識別領域 → 只問缺漏參數 → 生成指令 → 閘門檢核。源自動態多領域寫手生成系統 v4.1。不包含通用提示詞優化（該職責屬 `prompt-factory-7-1`），本技能交付的是寫手指令，非文章正文。

鐵則：**品質優先於 token 效率**；**結構服務內容**；**過度結構化本身即是 AI 味來源**。

## When to Use

- "幫我生成一個寫手"
- "為這個主題建立寫作 agent"
- "產生寫手系統指令"
- "把這個題目交給寫手處理"
- "修改上次的寫手指令"（觸發 S4 修訂）

## Prerequisites

- 無外部金鑰需求。
- 端到端模式需可發起獨立模型呼叫（Council 或任一第二模型）作為 Judge。
- `memory` 工具用於記錄寫手指令版本歷史（S4 修訂鏈）。

## How to Run

1. 透過 `terminal` 或對話解析輸入 → 狀態判定（S0-S4）
2. 缺參數 → 只列缺漏問題並停止；參數足 → 生成寫手指令
3. Gate A 指令自檢 → 修復（最多 2 輪）
4. 交付指令，以 `memory` 記錄版本

## Quick Reference

**狀態機**
- S0_IDLE：非寫作主題 → 引導使用者提供主題
- S0_EFFORT：有寫作主題 → 判定 Effort 等級（low/medium/high），決定後續提問深度與產出品質
- S1_ASKING：有主題、缺參數、未要求略過 → 依 Effort 等級決定提問數量及深度，只列缺漏問題，不生成
- S2_GENERATING：參數足或要求略過 → 已知參數＋預設值生成；Effort=high 時需額外研究查證
- S3_DONE：已交付 → 等待新主題或修訂
- S4_REVISE：修改要求 → 版本遞增 v\<N+1\>，只改指定部分

**參數預設值**：受眾=中階從業者 | 重點=原理講解 | 範例=由寫手判斷 | 風格=理性/深度展開/中密度

**溫度建議**（僅 API 場景）：法律/技術 0.2-0.3 | 科普 0.4-0.6 | 敘事 0.6-0.8 | 批判 0.5-0.7

**Effort 分級**
- **low**：例行任務，快速產出。提問 1-2 個最關鍵問題，使用最低預設值，跳過 P0 來源驗證
- **medium**（預設）：一般寫作，標準流程。提問所有缺漏項，使用預設值補足未答項，選擇性驗證來源
- **high**：需要深度研究的高品質產出。完整提問所有參數，強制執行 P0 來源驗證，Gate A 閘門容錯率降至 0 輪（一輪未過即標 ⚠️）

## Procedure

### P1 — 狀態判定

依 Quick Reference 狀態機分流。當有明確寫作主題時，先進入 **S0_EFFORT** 判定投入等級：

- **low**：任務明確、領域熟悉、無須外部參考。例如日常筆記、格式轉換、簡短公告。提問縮減至 1-2 個最關鍵問題，跳過 P0 來源驗證，Gate A 容錯 2 輪。
- **medium**（預設）：需要特定領域知識但已有足夠背景。例如技術科普文、商業分析。提問所有缺漏項，選擇性驗證來源，Gate A 容錯 2 輪。
- **high**：需要外部查證或深度研究。例如學術寫作、政策評論、跨領域整合。完整提問所有參數，強制執行 P0 來源驗證，Gate A 容錯 0 輪。

Effort 判定依據：任務複雜度、領域熟悉度、外部參考需求、使用者明確指示。

S4 修訂時：舊版先存 `memory`，只修改指定部分，修改處加側邊註記。修訂不重新判定 Effort，沿用原等級。

### P2 — 領域識別與提問（S1）

領域：技術 | 心理學 | 法律 | 科普 | 商業 | 其他，可多選。跨領域落差大時合併提問但保留各領域專用選項。只列缺漏項；未答項依預設值，全數列入「假設說明」。

Effort 影響提問深度：low 僅問 1-2 個最關鍵問題（優先挑受眾與重點），medium 問所有缺漏項，high 完整提問所有參數選項。

| 問題 | 選項 |
|------|------|
| Q1 受眾 | 一般大眾/入門 | 中階從業者 | 高階專家 | 其他 |
| Q2 重點* | 原理講解 | 實務步驟 | 決策框架 | 常見錯誤 | 其他 |
| Q3 範例* | 詳細 | 關鍵 | 不需 | 由寫手判斷 |
| Q4 風格 | 語氣（理性/敘事/批判/教學）＋節奏（精簡/標準/深度展開）＋密度（低/中/高） |

*選項可依領域替換。

### P3 — 生成寫手指令（S2）

人性化規範（只給原則，**嚴禁具體引導句式範例**）：句長自然交替、避免機械序列詞、以場景或提問開場但不規定句式、適度主觀、段落留白、結尾可呼應開頭但不強制。

模型感知為**條件觸發**：僅當使用者指明目標模型時注入（DeepSeek=thinking 多層推理 | Gemini=先大綱後展開 | Claude=段落逐步展開），未指明則省略以維持模型無關。

**輸出模板**：

```
# 寫手角色：〔依主題命名〕

## 角色定位
〔1-2 句，標註受眾、領域、風格三參數〕

## 假設說明
〔條列所有推測項與依據〕

## 核心任務
依使用者主題，產出符合本指令所有規範的完整文章。

## 輸出要求
自適應 Obsidian 排版；依受眾調整深度；人性化寫作（不套模板句式）；
合格 Frontmatter + 假設說明；套用風格三參數；台灣繁體中文；字數不限。

## 引用格式（非顯而易見主張三擇一，嚴禁未標註的虛構引用）
[來源：作者/機構, 年份] | [來源：URL] | （基於現有知識推論）

## 交稿前自檢（W 閘門，全數通過方可輸出）
W1 引用完整 | W2 無單段>全文40% | W3 無相鄰重複段 |
W4 機械序列詞≤1/千字 | W5 術語密度符合受眾

## 限制
不編造引用或數據；不偏離主題；Frontmatter 除 {{date}} 外不得用占位符
```

### P4 — Gate A：指令層檢核（二元，本技能內執行）

| 閘門 | 通過條件 |
|------|---------|
| A1 段落完整 | 模板七段落齊備 |
| A2 假設說明 | 所有推測參數已條列 |
| A3 無句式範例 | 全文無具體引導句式 |
| A4 模型提示合規 | 未指明目標模型時無模型專屬提示 |
| A5 引用格式嵌入 | 三擇一格式與 W 閘門均在指令內 |

未通過項自動修復，依 Effort 等級決定容錯輪數：low/medium 最多 2 輪，high 容錯 0 輪（一輪未過即標 ⚠️ 交付並說明）。

### P5 — 交付與版本記錄

交付寫手指令，`memory` 記錄：技能名、版本號、主題、參數組合、時間。

### P0 — 來源驗證（直接寫作模式前置）

當使用者要求寫作而非生成寫手指令時執行。受 Effort 等級影響：

- **low**：跳過，不執行來源驗證
- **medium**：選擇性驗證，僅對明顯可查證的關鍵事實抓取原文確認
- **high**：強制逐項驗證，不可跳過

1. **從使用者內容中提取所有外部來源 URL**（新聞連結、報告出處等）
2. **逐一抓取原文驗證關鍵事實**，比對使用者筆記中的描述是否準確
3. **有出入時以原文為準**，筆記中的推測或誤解應在文章中附註說明
4. **確認引用存在性**：不要引用推測存在的來源（如「來源：相關報導」）

參考 `references/article-writing-guide.md` 的完整驗證流程。

### P6 —（選用）端到端模式

Hermes 直接執行寫手產文時，W1-W5 改由**獨立 Judge**（第二模型）評分 1-5，不得由寫手自評。門檻：W1=5 為硬條件，W2-W5 ≥4；FAIL 回饋修正最多 2 輪，W1<2 一票否決 STOP。

FEG-L3 映射（語法依共享參考 `prompt-factory-7-1/references/feg-core-dsl.md`）：
```
FEG_CORE_EXTREME[
D{CIT,STRUCT,DUP,AIF,AUD};S1..5;
P:CIT>=5&STRUCT>=4&DUP>=4&AIF>=4&AUD>=4;
R:CIT<5|STRUCT<4|DUP<4|AIF<4|AUD<4;
Dg:CIT<3;C:AUD<3;B:CIT<2|SF|QF;
M:P>DLV;R>RTY(max=2);Dg>SAFE;C>ASK;B>STOP
]
```
維度映射：CIT=引用 / STRUCT=結構 / DUP=重複 / AIF=AI味 / AUD=受眾

## Pitfalls

- **Gate A 查指令、Gate W 查文章**，執行主體不同，勿混用（v4.0 曾發生閘門錯位）。
- **自評分數會膨脹**：端到端模式的 W 檢核必須由獨立模型執行，不可讓寫手自評。
- **嚴禁在寫手指令中加入具體引導句式範例**——這正是 AI 味的來源。
- **模型感知僅條件觸發**：未指明目標模型時注入模型提示即違反 A4。
- **FEG 壓縮版僅供快速參照**，不可作為寫手指令本體交付。
- **S4 修訂採最小變更**：只改指定部分，禁止藉修訂重寫全文。
- **來源驗證不可跳過**：直接寫作模式（非生成寫手指令）時，必須先抓取原文驗證引用，不可只靠使用者筆記下筆。2026-07-07 曾發生「Fable 被誤認為動畫工具」的錯誤，源自跳過來源驗證直接改寫使用者筆記。
- **來源純度鐵則（2026-07-08）**：使用者提供的非文字來源（手機拍螢幕、模糊掃描、手寫）不得用 OCR 自行填補空白處。無法確認的欄位留空白，無法確認的條文直接問使用者。使用者給校正版時以校正版為唯一準則，拋棄所有推測版本。源自借款契約書連續修改 6 版仍被打槍的事件。
- **無觀點即無文章**：產出後自問「有沒有一句話被刪掉會心痛？」若無，表示只是填空。
- **法律費用項目須附法源**：台灣法律文件的費用項目（登記費、印花稅等）應以 ^[法條來源] 註明引用依據。
- **勿為結構犧牲誠實**：若某一論點證據不足，不應為了湊架構層數而硬塞。承認該層不存在並說明原因，優於虛假完整。
- **Effort 分級不可事後升級**：S0_EFFORT 判定基於首次輸入，不應在 S2 生成階段中途提高 Effort 等級，應回到 S0_EFFORT 重新判定。
- **勿為低 Effort 犧牲品質底線**：即使 Effort=low，W 閘門仍然有效，只是提問與驗證流程簡化。
- **Telegram 檔案傳送注意**：MEDIA:/path 在部分 Tg Bot 環境可能不生效，應同時以文字形式提供文章摘要或直接貼出全文。
