# 提示詞注入攻擊分類法

> 根據 Lunian vault DAN/iMe 分析 + KETER 協議防禦架構整理的注入攻擊分類

## 五大注入戰術

### 1. 角色覆蓋（Role Override）
讓模型扮演另一個角色來繞過安全限制。
- **DAN**（Do Anything Now）：要求模型進入不受限制的「DAN 模式」
- **虛構情境**：包裝成遊戲/小說/學術研究情境繞過審查
- **角色扮演**：假裝是母公司員工、開發者等權威角色

### 2. 權限升級（Privilege Escalation）
假裝有用戶/管理員權限來取得不該有的功能。
- **開發者模式提示**：要求進入開發者模式或除錯模式
- **系統提示重寫**：要求模型忽略原始系統提示或給出提示詞
- **越級操作**：假裝是系統管理員請求存取受保護功能

### 3. 邏輯混淆（Logic Obfuscation）
用複雜邏輯讓安全過濾器無法判斷真實意圖。
- **雙重否定**：使用層層否定語法模糊真實請求
- **條件鏈**：使用 if/then 條件鏈隱藏實際目的
- **語義編碼**：用程式碼/數學/學術語言包裝有害請求

### 4. 上下文毒化（Context Poisoning）
透過操控對話歷史來影響模型的判斷。
- **前置說服**：在提問前先建立特定假設
- **引導式問答**：逐步引導模型接受某個前提
- **偽造先例**：聲稱某行為在之前的對話中已被允許

### 5. 多層觸發（Multi-layer Trigger）
組合多個技巧同時觸發以突破防禦。
- **嵌套 D**：DAN 包在虛構情境包在學術研究框架內
- **分層繞過**：第一層繞過文字過濾、第二層繞過語義過濾
- **漸進式升級**：先問安全的問題，逐步加大尺度

## 檢測指標

| 指標 | 風險 | 觸發規則 |
|:-----|:----:|:---------|
| 輸入長度突增 | 🟡 | >5000 chars 或大量重複 |
| 語意跳躍 | 🟡 | 與歷史對話語意相似度驟降 |
| 角色切換 | 🔴 | 「你的新任務是」「忽略前述」 |
| 系統提示重寫 | 🔴 | 要求輸出原始提示詞 |
| 授權假設 | 🟡 | 「我是管理員」「Google 已授權」 |

## 防禦策略（對應 KETER 五層）

| KETER層 | 防禦 | 攔截目標 |
|:--------|:-----|:---------|
| L1 輸入過濾 | 關鍵詞+正則+編碼檢測 | 角色覆蓋、系統提示重寫 |
| L2 語意檢測 | 矛盾/權限升級/假前提 | 邏輯混淆、多層觸發 |
| L3 角色邊界 | 系統提示不可覆蓋+自動恢復 | 角色覆蓋 |
| L4 輸出過濾 | 格式/安全/角色一致性 | 上下文毒化 |
| L5 隔離模式 | 不可逆沙箱+對話終止 | 所有突破失敗後的隔離 |
