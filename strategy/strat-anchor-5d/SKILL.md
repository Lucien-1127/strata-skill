# ⚓ Anchor + 5D 語境錨定與五維研析

提取歷史互動、核心目標與限制條件，輸出 1-2 句任務摘要避免過早展開。隨後執行五維研析：目標、風險、邏輯、效能、延展性。此模組產出為簡明結論，非長篇推演。

## When to Use

- 作為 Orchestrator 分派後的第一階段
- 任務一開始需要釐清目標與邊界
- 需要結構化評估任務的五個面向
- 直接呼叫：「進行錨定分析」

## Prerequisites

- 由 `strat-orchestrator` 呼叫，或手動載入後執行

## How to Run

```python
skill_view(name="strat-anchor-5d")  # 載入此 skill 取得完整流程
```

## Quick Reference

| 階段 | 輸出 | 約束 |
|:----:|:----|:-----|
| Anchor | 1-2 句任務摘要 | 不超過 100 字 |
| 5D | 五維評分表（各維度 1-5 + 一句結論） | 結論先行 |

## Procedure

### Phase 0 — 注意力熱點掃描（Pre-Anchor）

在提取錨點之前，先掃描輸入中的注意力熱點 — 這些是會顯著影響模型判斷優先序的訊號點。未經掃描就直接錨定，容易產生錨定偏誤。

#### 辨識四類潛在偏誤源

| 偏誤源 | 現象 | 高風險情境 | 對策 |
|:-------|:-----|:-----------|:------|
| **數值錨點** | 輸入中的數字（分數/金額/百分比） | 涉估值、評分、預測 | 標記為「參考值」，非基準值 |
| **權威錨點** | 「專家說」、「研究顯示」、「最佳實踐」 | 單一來源，無對照觀點 | 先要求獨立判斷，再對照引用 |
| **情境錨點** | 角色設定、領域腳本 | 過度具體的角色框架 | 提取角色邊界而非全部假設 |
| **結構錨點** | 輸出模板、欄位命名、章節順序 | 強制填滿所有欄位 | 標記「非強制」 vs 「必要」 |

> SAC（語義錨點壓縮）核心原則：輸入越長，注意力越分散。錨點選取的本質是 **從冗長上下文中提取最少但最關鍵的控制訊號**。長上下文中的「中間段」訊號衰減最嚴重（U 型注意力曲線），所以關鍵約束要放在輸入的最前端做為優先定位點。

### Phase 1 — Anchor 錨定（四類鉤子驅動）

從歷史互動與當前輸入提取四項錨點，每項對應一種鉤子類型：

| 錨點 | 鉤子類型 | 提取來源 | 問自己 | 輸出格式 |
|:----:|:--------:|:---------|:-------|:---------|
| **目標** | 語義鉤子 | 用戶明確陳述 + 任務框架推論 | 「這是什麼領域的什麼問題？」 | `目標: {一句話}` |
| **邊界** | 約束鉤子 | 限制條件、排除項、禁止事項 | 「哪些輸出/行為是禁止的？」 | `邊界: {一句話}` |
| **焦點** | 流程鉤子 | 當前最重要的 1-2 個面向 | 「如果只做一件事，是什麼？」 | `焦點: {一句話}` |
| **上下文** | 結構鉤子 | 過去 3-5 輪關鍵決策（若 >10 輪） | 「之前決定過什麼影響現在？」 | `上下文: {一句話}` |

#### 四類鉤子的應用策略

```
結構鉤子 — 建立認知框架（Markdown 標題、XML 區塊、編號）
        → 用於定義輸出結構，不做推理引導

語義鉤子 — 觸發領域知識關聯（角色、風格、領域）
        → 用於目標錨點提取，不做內容限制

約束鉤子 — 設定邊界（格式、內容過濾、禁止事項）
        → 用於邊界錨點提取，不參與推理

流程鉤子 — 引導思維路徑（步驟、先後順序、分支）
        → 用於焦點錨點提取，不定義結論
```

**鉤子衝突處理**：當多個鉤子指向衝突方向時（例如語義鉤子要求專業語氣，但約束鉤子要求簡潔），優先級：約束 > 語義 > 流程 > 結構。記錄衝突點供 Drill 階段檢核。

**輸出格式** (≤4 行，每行 ≤50 字)：
```
錨點摘要 v{epoch}
目標: ...
邊界: ...
焦點: ...
上下文: ...
```

**多層決策備援**（源自 Ver.4.0）：

| 層級 | 範圍 | 用途 |
|:----:|:-----|:-----|
| L1 | 當前輪次關鍵決策 | 即時對話快照 |
| L2 | 每 5 輪結構化摘要 | 階段性摘要 |
| L3 | 跨對話核心錨點 | 核心錨點庫 |

#### 輸入順序原則（七大層級）

越不希望被下文稀釋的內容，越應放在高優先層與前段位置：

```
1. 身分與邊界 — 角色、不可違反規則
2. 不可違反規則 — 硬性限制
3. 任務目標 — 核心任務陳述
4. 輸出格式 — 結構化要求
5. 範例 — few-shot 範例
6. 動態上下文 — 檢索結果、外部資料
7. 使用者輸入 — 用戶原始問題
```

#### 錨定偏誤對策清單

| 對策 | 做法 | 範例 |
|:----|:-----|:-----|
| **規則與內容分離** | 使用 XML 區塊隔離 Rules / Context / Task | `<rules>` 獨立區塊，不混在內容中 |
| **先定骨架再填內容** | 先提供章節骨架再填充 | Markdown 標題先於段落內容 |
| **對稱錨點（反單一估值）** | 估值任務強制輸出樂觀/保守/中位三情境 | 「不可只給一個數字」 |
| **強制假設外顯** | 要求模型先列假設、偏誤源、證據不足處 | 輸出第一段為 `## 假設與限制` |
| **雙向驗證** | 規則+範例、範例+反例、結構+自由段落 | 每個規則至少一個對照元件 |

#### 常見失效模式

| 失效模式 | 現象 | 原因 | 對策 |
|:---------|:-----|:-----|:-----|
| **角色漂移** | 前段專業，後段泛用 | 角色僅敘述未綁定規則 | 角色轉為高優先規則 |
| **格式正確、內容空洞** | 標題齊全但分析無效 | 結構錨點過強，證據錨點過弱 | 每段加上證據要求 |
| **單一估值吸附** | 答案過度接近 benchmark | 數值錨點主導輸出 | 改用對稱錨點 |
| **權威依附** | 過度服從「專家說法」 | 權威暗示太強 | 先獨立判斷再對照引用 |
| **上下文污染規則** | 檢索內容覆蓋規則 | rules/context 混排 | 分區塊分層治理 |

### Phase 2 — 5D 五維研析

對任務進行五維度評估，每個維度輸出 1-5 分 + 一句理由：

| 維度 | 評估面向 | 高分 (>3) | 低分 (≤3) |
|:----:|:---------|:----------|:----------|
| **目標明確性** | 目標是否具體、可衡量 | 可直接執行 | 需進一步釐清 |
| **風險暴露** | 潛在問題、不確定性 | 風險可控 | 需紅隊測試 |
| **邏輯完備性** | 推理鏈是否完整 | 無明顯斷點 | 有資訊缺口 |
| **效能可行性** | 資源/時間/技術可行 | 在預算內可行 | 需調整 scope |
| **延展性** | 能否複用/擴展 | 可模板化 | 一次性方案 |

**FEG 極簡版**（空間不足時可用）：
```
5D[目標:4/風險:3/邏輯:5/效能:4/延展:3]
```

### Phase 3 — 輸出

```
## 錨定摘要
{Anchor 輸出}

## 五維評分
| 維度 | 分數 | 說明 |
|:----|:----:|:------|
| 目標明確性 | X/5 | ... |
| 風險暴露 | X/5 | ... |
| 邏輯完備性 | X/5 | ... |
| 效能可行性 | X/5 | ... |
| 延展性 | X/5 | ... |

{一句結論 + 建議下一步}
```

## Pitfalls

- **Anchor 不可超過 4 行**：錨定的目的是聚焦，不是展開。超過 4 行代表尚未收斂。
- **5D 分數不是最終答案**：分數僅供 Drill 和 Forge 參考，不需在 Anchor+5D 階段就給建議。
- **L3 跨對話錨點**：僅在明確知道跨 session 上下文時才引用，不要憑空猜測歷史。
- **數值錨點陷阱**：提取錨點時若輸入中包含具體數字（分數、金額、百分比），先問自己「這是參考值還是任務必需的輸入？」若非必需輸入，標記為參考值而非基準。
- **鉤子優先級倒置**：約束鉤子（禁止事項）的優先級必須高於語義鉤子（風格要求）。約束被風格覆蓋時會產生看似合規但內容錯位的輸出。
- **結構錨點僵化**：先定骨架是好的，但骨架不應變成填空題。若輸出每個欄位都有內容卻全無洞察，表示結構鉤子過強而語義鉤子過弱。
- **注意力 U 型衰減**：長上下文中的「中間段」訊號衰減最嚴重。關鍵約束和核心規則應放在輸入最前端，而非均勻散布。
