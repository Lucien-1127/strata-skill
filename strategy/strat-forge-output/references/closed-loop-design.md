# 提示詞工程閉環設計

> 來源：老闆 2026-03-21

## 閉環五環節

1. 輸入定義 → 2. 執行層 → 3. 評估層 → 4. 修正層 → 5. 再部署層

關鍵：**回饋必須能反向改變系統行為，否則只是監控。**

## 三層閉環

| 層級 | 範圍 | 對應 |
|:----:|:------|:------|
| 提示層 | wording、結構、few-shot | strat-forge-output 模板 |
| 流程層 | 步驟拆分、代理順序、重試 | strat-orchestrator 路由 |
| 治理層 | 版本、變更原因、回歸測試 | strat-forge-output S4_REVISE |

## 輸出契約

一份實用的輸出契約應包含：欄位結構、格式要求、禁止事項、最低品質門檻、失敗回退格式。→ 對應 strat-forge-output Output Spec

## 錯誤分類法

格式錯誤 / 任務理解偏差 / 上下文不足 / 過度推論 / 違反限制 / 漏答欄位 / 代理交接失真

→ 對應 strat-forge-output 品質閘門 G1-G5

## 版本治理

應版本化的七項：Prompt本體、任務模板、模型版本、參數設定、評估規則、few-shot版本、路由配置。
