## RBAC vs ABAC

RBAC 適合固定角色，ABAC 適合動態判斷，趨勢是 Policy-as-Code。

## OPA 範例 Rego

```rego
package authz
default allow = false
allow if { input.user.role == "admin" }
```

# 提示詞鉤子：結構化設計指南

> 來源：老闆知識庫

## 四大鉤子類型

| 類型 | 核心機制 | 適用場景 |
|:----:|:---------|:---------|
| **結構鉤子** | 建立認知框架（Markdown 標題、編號、XML 標籤） | 複雜任務拆解、多步驟流程 |
| **語義鉤子** | 觸發特定知識關聯（角色扮演、風格模仿） | 領域專家模擬、語氣控制 |
| **約束鉤子** | 設定邊界與限制（格式、內容過濾） | 避免幻覺、輸出格式控制 |
| **流程鉤子** | 引導思維路徑（思維鏈、逐步分析） | 多步驟推理、反思修正 |

## 提示架構四層次

1. 角色與背景層 — 語義鉤子
2. 任務目標層 — 結構鉤子
3. 思維流程層 — 流程鉤子
4. 輸出約束層 — 約束鉤子

## 對應 strat 系統

- 結構鉤子 → strat-forge-output 格式規範
- 語義鉤子 → strat-orchestrator 角色切換
- 流程鉤子 → strat-anchor-5d 分析流程
- 約束鉤子 → strat-forge-output Output Spec
