# AI 對話穩定系統方案

> 來源：老闆 2026-05-22

## 三大痛點

| 問題 | 症狀 | 根本原因 |
|:----:|:-----|:---------|
| 長對話失憶 | >15 輪忘記設定 | 靜態人設注意力衰減 |
| 狀態校驗失效 | 前後矛盾 | 無狀態機機制 |
| 流程控制混亂 | 隨意跳步 | 無流程守門(Gate) |

## 三層穩定控制體系

### Layer 1 — 動態狀態機

將靜態人設轉為 AI 主動維護的動態變數：

```yaml
state_variables:
  tone_cynicism: 8/10
  task_progress: "需求釐清階段"
  confirmed_facts: []
```

### Layer 2 — 記憶護欄機制

| 層級 | 位置 | 目的 |
|:----:|:----:|:-----|
| 長期記憶 | 上下文開頭 | 背景框架 |
| 短期記憶 | 用戶輸入前方 | 維持連貫性 |
| 護欄規則 | **緊貼當前輸入** | 🔒 防止走鐘 |

### Layer 3 — 流程閉環控制

每輪輸出前強制自我校驗。

## 對應 strat 系統

- Layer 1 → strat-memory-policy 核心分級
- Layer 2 → strat-memory-policy 護欄規則
- Layer 3 → strat-forge-output 品質閘門
