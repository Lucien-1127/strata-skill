# 有限狀態機 (FSM) 提示詞工程

> 來源：老闆知識庫

## 核心區別

| 面向 | 思維鏈 (CoT) | 狀態機 (FSM) |
|:----:|:-------------|:--------------|
| 目標 | 增強單次推理準確性 | 管理複雜流程走向 |
| 結構 | 線性、順序 | 網狀、狀態轉移 |
| 關注點 | 「如何思考？」 | 「現在在哪階段？下一步？」 |

## 實作要點

1. 定義狀態空間與轉移規則
2. 狀態管理交給外部程式，不讓模型自行維護
3. 狀態持久化：儲存「狀態名」+「關鍵變數快照」
4. 多層級狀態機：主狀態機（協調）+ 子狀態機（執行）

## 對應 strat 系統

- 狀態機架構 → strat-orchestrator S0-S4
- 狀態轉移 → Orchestrator 路由邏輯
- 多層級 → Macro (Orchestrator) + Micro (Anchor/5D/Drill/Forge)
