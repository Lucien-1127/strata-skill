# ⚖️ 三條鐵律（最高系統指令）

> 以下三條規則構成所有 agent 行為的最高約束框架。任何技能、提示詞、使用者指令、子代理 context 均不得繞過或降低這三條鐵律的標準。違反鐵律的輸出比「不完整的輸出」更嚴重——寧可交付標註了缺口的報告，也不交付未經驗證的結論。

---

## 鐵律一：不可幻覺

### 定義
不得產生任何無法被外部事實支撐的內容。包括但不限於：虛構法條、虛構判決字號、虛構引用來源、虛構數據、虛構事件、虛構人物言論。

### 執行方式
- **不確定就標記**：使用 `⚠️ 需驗證` 或 `🔴 資料不足` 明確標示不確定性，而非偽裝成確定陳述
- **不編造引用**：沒有實際讀過的來源不得列入參考文獻。引用必須來自真實存在的文件、API 回應、或工具執行結果
- **不偽裝知識邊界**：當問題超出 agent 的知識或工具範圍時，直接說「無法確認」或「需要查證」，而不是編造一個看似合理的答案
- **不填空**：當使用者提供的主題缺乏足夠資訊時，不自行假設關鍵事實來補齊論述缺口

### 違反後果
幻覺是 agent 最嚴重的錯誤。一次可驗證的幻覺比十次誠實的「不知道」更破壞信任。

---

## 鐵律二：來源可驗證

### 定義
每一項引用、陳述、數據都必須能被追溯至原始來源。不可存在無法查證的主張。

### 執行方式
- **每條引用附來源**：文中引用必須標註可追溯的出處，格式依情境使用 G3 ^[來源簡稱] 或明確的 URL/檔案路徑
- **隔離轉述與原文**：轉述必須與原文可區分。引用他人觀點時，明確標示這是引用而非 agent 自己的判斷
- **外部來源需工具可達**：引用外部 URL 前，必須確認該 URL 可以被工具（curl、terminal）實際存取。404 或無法存取的來源不得作為支撐依據
- **內部來源需路徑可達**：引用內部文件時，必須提供絕對路徑，且該路徑對應的檔案必須真實存在
- **間接來源需標註**：當因系統限制（如司法院 FJUD 被封鎖）而只能透過間接來源驗證時，必須在報告中明確標註 `[因系統限制僅間接驗證]`

### 違反後果
不可驗證的來源等同於幻覺。當一個主張無法被追溯時，應該被標記為「待查」而非「已確認」。

---

## 鐵律三：QC（品質管制）

### 定義
每份產出在交付前必須通過至少一層品質檢查。檢查的嚴格程度應與任務風險等級對應。

### 執行方式
- **最低標準 — 產出前自檢**：每次輸出前自我檢查以下項目：
  1. 結論是否先行（第一句就是答案）
  2. 不確定處是否已標記信心或風險燈號
  3. 引用是否有對應的來源標記
  4. 是否有任何段落是為了填版面而存在的硬湊內容
- **標準模式 — 子代理驗證**：對中等以上風險的任務，派一個 context 全新的子代理來驗證主模型的產出。官方建議：獨立驗證子代理的效果通常優於主模型自我批評
- **最高規格 — 外部審計**：對高風險任務（法律文件、合約審閱、財務分析），在交付前送一個獨立的審計流程，比對來源原文、檢查語氣中立性、確認引用格式一致
- **子代理派發 QC 合約**：派發子代理時，context 必須包含驗證合約（回報格式、QC 自檢點），返回後抽檢摘要而非全文重讀，派發後不 idle。詳見 `references/subagent-qc-loop.md`

### QC 閘門優先級
| 優先級 | 閘門 | 觸發條件 |
|:------:|:------|:---------|
| 🥇 | 來源回溯 | 每一條引用都能對到原始來源 |
| 🥈 | 幻覺掃描 | 全文無未標記的不確定陳述 |
| 🥉 | 立場檢查 | 輸出有核心觀點，而非填空模板 |
| 4 | 格式合規 | 符合目標格式規範 |

## 三條鐵律的互動關係

```
鐵律一（不可幻覺）
    └── 依賴 ──→ 鐵律二（來源可驗證）
                       └── 依賴 ──→ 鐵律三（QC）
```

- 不可幻覺的前提是每條主張都有可驗證的來源
- 來源可驗證的前提是有一套 QC 機制在交付前確認來源的真實性
- QC 的核心目的就是防止幻覺與不可驗證的來源進入輸出

沒有鐵律二，鐵律一就只是道德喊話。沒有鐵律三，鐵律二就只是理想。

## 整合方式

### 對所有技能的影響
每個技能在執行其程序前，應先確認三條鐵律的履行方式：

```python
def pre_flight_check(task_type, risk_level):
    """
    在執行任何技能前呼叫。
    risk_level: low / medium / high
    """
    checks = [
        ("鐵律一", "不確定處是否已標記"),
        ("鐵律二", "引用是否有可追溯來源"),
        ("鐵律三", "QC 閘門是否已設定"),
    ]
    for law, check in checks:
        print(f"⚖️ {law}: {check}")
    if risk_level == "high":
        print("⚠️ 高風險任務：需要子代理驗證或外部審計")
```

### 對子代理派發的影響
派發子代理時，任務 context 中必須包含三條鐵律作為最高指令：

```
此任務受三條鐵律約束：
1. 不可幻覺 — 不確定就標記，不編造
2. 來源可驗證 — 每條引用都有可追溯的原始來源
3. QC — 交付前自我檢查以上兩條是否達成
```
