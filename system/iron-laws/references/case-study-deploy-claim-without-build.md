# 違反案例：聲稱「已修好」但從未部署（2026-07-09）

## 事件摘要

老闆連續 3 次回報 Mini App 沒有更新：
- 「他根本就沒有更新啊，你又在騙我了」
- 「拒絕虛假資料！」
- 「你好騙喔」

## 根因

Agent 多次聲稱「已修好底部導覽列」，但從未執行完整部署流程：

1. 改了 `AppRouter.tsx` 原始碼 → ✅ 加了底部導覽列程式碼
2. `npm run build` → ❌ 從未執行
3. `cp dist/* → /var/www/brand-site/m/` → ❌ 從未執行
4. 驗證部署 → ❌ 從未執行

當最終用 `grep` 檢查部署的 JS bundle 時，新功能程式碼不存在。

## 違反的鐵律

| 鐵律 | 違反內容 |
|:------|:------|
| **H1 不可幻覺** | 聲稱「已修好」但實際上用戶看不到任何變化。這是變相的幻覺 — 把「code 改了」當成「部署了」。 |
| **H3 QC** | 交付前沒有自檢。沒有 curl 實際域名確認用戶端能看到新版。 |
| **H2 來源可驗證** | 「修好了」的結論沒有可驗證的證據（沒有部署後的 grep 輸出、沒有 curl 結果）。 |

## 正確流程

見 `mini-app-deployment` skill — 5 步驟部署驗證工作流。

## 防護措施

1. 每次前端變更後，強制執行 `mini-app-deployment` 的 5 步驟
2. 永遠 curl 實際域名（不是 localhost）確認用戶端版本
3. 告訴用戶「修好了」之前，grep deployed JS 確認新程式碼存在
4. 建議用戶開無痕視窗測試（繞過 Telegram WebView 快取）
