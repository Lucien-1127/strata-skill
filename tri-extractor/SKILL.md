# ⚡ 三合一萃取器

## 觸發條件
用戶在 Telegram 提到「摘要這篇」、「整理重點」、「畫心智圖」、「萃取內容」或傳送長文連結時自動觸發。

## 功用
將任何長文/筆記/文章/對話記錄，以極簡格式輸出三種形式之一：
- **摘要** — 3 點核心，每點 ≤20 字
- **重點** — 3 點關鍵，每點 ≤20 字
- **心智圖** — Mermaid mindmap，三層結構

## 使用方式
跟我說：
- 「摘要這篇：{貼上內容}」
- 「整理重點：{貼上內容}」
- 「畫心智圖：{貼上內容}」

或直接傳送 Telegram 訊息/連結，我會自動判斷。

## 執行步驟

### 步驟 1：解析用戶意圖
- 用戶說「摘要」→ 摘要模式
- 用戶說「重點」→ 重點模式
- 用戶說「心智圖」→ 心智圖模式
- 無指定 → 預設「摘要」

### 步驟 2：萃取內容
如果用戶只貼了網址，先用 web_extract 抓取內容。

### 步驟 3：執行萃取
套用以下提示詞邏輯（隱式執行，不輸出提示詞本身）：

**摘要 / 重點模式：**
- 輸出恰好 3 個項目符號（`- `）
- 每點 ≤20 字
- 無空行、無修飾、零廢話

**心智圖模式：**
- 輸出 Mermaid mindmap 程式碼塊
- 根節點 `root(主題)`，最多三層
- 子節點 ≤4 字

### 步驟 4：回傳
直接輸出結果，不附加任何說明。

## 與原版 Copilot 提示詞的差異

| 面向 | 原版（Obsidian Copilot） | 轉換後（Hermes Skill） |
|------|------------------------|----------------------|
| 使用場景 | 僅限 Obsidian 內 `/slash` | Telegram / CLI / Web 皆可 |
| 輸入方式 | 手動選取內容 + 輸入指令 | 直接對話 / 貼連結自動抓 |
| 觸發方式 | 需用戶主動執行 | Hermes 自動偵測意圖觸發 |
| 擴充能力 | 靜態提示詞 | 可串接其他 skill（翻譯→萃取→存檔） |
| 內容源 | 僅限 Obsidian 當前筆記 | 網頁/訊息/文件/對話皆可 |
| 輸出目標 | 螢幕顯示 | 可自動存 Obsidian / 傳回 Telegram |
