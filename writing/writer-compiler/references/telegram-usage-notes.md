# 實戰經驗記錄

## Telegram 環境的退回機制

`ask_user_input_v0` 按鈕工具僅在 Hermes Web UI / 支援按鈕的環境可用。
在 Telegram 純文字環境下，自動退回數字編碼問法：

```
Q2 劇本重點？
① 搞笑橋段設計
② 角色動作編排
③ 場景轉場腳本
④ 混合

回覆格式：Q2=①
```

### 效率問題
- 缺漏 5-6 題需分兩輪問，在 Telegram 文字對話中流程較長
- 使用者明確給出「快速/直接生成/不用問」時，可直接跳 S2 智慧預設

### 未定義領域的處理
當主題不屬於六大領域（技術/法律/心理學/科普/商業）時，歸類為「跨領域」。
例如動畫劇本、創作類內容 → 跨領域。D13（分領域說明）為主推項。

### 使用者偏好
- 此使用者（Boss）偏好高效流程：若主題明確、參數可合理推測，直接跳 S2 生成比逐步提問更符合需求
- 但初次使用本技能時，遵循完整 S0→S1→S1.5→S2 流程可驗證技能完整性
