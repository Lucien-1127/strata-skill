# 寫手指令編譯參考(v3.5)

S2_GENERATING 必讀。依序套用:人性化原則 → 通用排版原則 → 措辭三軸 → 引用嚴格度檔位 → 閘門生成 →〔SERIES=true〕全系列規劃生成(含衝突強度分級) →〔SERIES=true〕故事人物設定生成 → 輸出模板(階層式控制架構:核心任務 →〔語流特徵干擾模組,隱式傾向,I3a–I3e 不展開〕→〔視角與細節錨定模組〕→ 生成前置校準〔內在獨白,防漂移〕→ 交稿前自檢〔I3a–I3e 唯一展開位置,隱式觸發〕)。

## 系列規劃生成(SERIES=true 專用)
依 S1B 系列規劃題答案(集數概估、大綱主軸走向)產出,順序不可顛倒:
1. **全系列規劃**:依集數概估逐集列出篇章序號、篇章名稱、一句話大綱(該集核心事件/轉折),整體須符合所選大綱主軸走向(漸進升級/多線匯流/懸疑揭露/單元＋暗線等)
2. **故事人物設定**:必須晚於①完成。為主要角色(依主題規模抓 2–5 位)定義:核心目標、核心衝突、貫穿弧線；弧線之每個轉折須標注對應①中的篇章序號,禁止脫離大綱憑空設定人物發展
使用者於 S1B 明示略過系列規劃題 → 兩段仍保留段落標題,內容以「使用者略過,建議續編前補齊」標註,不得省略段落本身。

## 人性化寫作原則(去 AI 味,創作時之軟性引導)
**嚴禁在寫手指令中提供具體引導句式範例**,只寫原則:
- 句長自然交替
- 以具體場景或提問開場,但不規定句式
- 適度主觀觀點
- 段落留呼吸空間
- 結尾可呼應開頭(意象/提問回扣),但不得重述本段或全文已陳述之論點——呼應與重述主旨的差異見 I3c

## 通用排版原則
標題層級 ≤3;粗體節制(每段 ≤1 處);段落間留空行;清單僅用於真正並列項。禁止 Obsidian 專屬語法(frontmatter、雙括號連結、callout、dataview)。

## 措辭三軸(依領域動態定調,寫入寫手指令)
- **精準度**:術語嚴格 ↔ 口語
- **比喻容忍度**:高 ↔ 低
- **情感濃度**:克制 ↔ 流露

示例(非封閉清單):法律/技術 → 高精準、低比喻、克制;科普 → 中精準、高比喻;敘事/人文 → 高情感、低結構約束;批判/評論 → 立場明確、論據紮實。

## 模型適配句(僅執行模型有指定時擇一注入)
- DeepSeek:善用多層推理逐步深化論證
- Gemini:先擬大綱再展開全文
- Claude:按段落逐步展開,善用長格式

## 引用嚴格度檔位(依領域與主軸動態選定,寫入引用規範)
- **高**(規格/法律/醫療/宣稱官方標準的教學):每項規格性主張須與來源實際內容一致;無法確認一致 → 該主張降級標註(基於現有知識推論),禁止掛來源背書
- **中**(一般技術/商業/科普):非顯而易見主張附來源或標推論
- **低**(敘事/人文/個人觀點):僅事實性錨點需標註,觀點免引

## 閘門生成規則(取代固定清單)

**不變式(I1–I2 + I3 家族,恆定寫入,禁止省略)**:
- I1 引用真實:所有引用主張與來源內容一致;無來源或無法確認 → 標「推論」。禁止虛構引用或數據
- I2 無重複:無相鄰段落表達同一概念
- I3a 機械序列詞(首先／其次／再者／此外／總而言之／值得注意的是／綜上所述／不可否認／毫無疑問地):每千字 ≤1 處
- I3b 萬能連接句式(不僅…而且…／無論…都…／從…到…):每千字 ≤2 處
- I3c 段落結尾重述主旨(結尾句以同義詞複述本段開頭已陳述之論點):全文 0 處。**與人性化原則「呼應開頭」之區別**:呼應可用意象、場景、提問回扣開頭,但不得換句話說再講一次論點
- I3d 連續段落開頭句式雷同(連續 ≥3 段以相同結構開頭,如皆以「在○○方面」「對於○○而言」起句):連續同構 ≤2 段
- I3e 句長變異度(連續 3 句字數皆相近,差距 <5 字,視為機械節奏):連續同幅 ≤2 句,需含至少 1 次顯著落差(短句 <15 字或長句 >40 字交替出現)。**強制/建議判定**:依編譯時已收集之措辭三軸「精準度」軸自動分流,不另行提問——精準度＝高(法律/醫療/規格類)→ 本項降級為「人性化原則」建議,不納入強制自檢;精準度＝中/低 → 本項納入強制自檢

**動態閘門(1–4 條,依主題/領域/主軸生成)**,每條須同時通過三判準:
1. **可客觀驗證**:通過與否可二元判定
2. **對應失效模式**:針對該文類最可能出錯處(例:規格教學 → 範例可直接執行、規格與官方文件一致;數據分析 → 每個數字可追溯;敘事 → 時間線與人物一致;決策文 → 每建議附否決條件)
3. **寫手可自行執行**:不依賴外部工具即可自檢

生成的閘門編號續接 G4 起,I1–I2 於「交稿前自檢」直接展開;I3a–I3e 採**隱式觸發**——生成過程中只給一句隱性傾向提示,不展開清單,唯一的完整展開位置在「交稿前自檢」草稿完成後核對(降低詞彙級約束常駐造成的品質稅);結構失衡(單段 >40%)、受眾校準、表現形式配比(敘事類之對話/敘述比例)屬常用動態項,依文類取用。

**約束-釋放對稱檢核**:閘門中任何「禁止 X 直到 Y」型規則,必須同步寫明 Y 狀態(高潮章/揭露點/解禁條件)之行為規範——約束的例外狀態不可留白。

## 寫手指令輸出模板

```markdown
# 寫手角色:〔依主題命名〕

## 角色定位
〔1–2 句專長,標註受眾、領域、風格參數〕

## 假設說明
〔條列所有推測項與依據〕

〔SERIES=true 時插入以下二段,位置在「假設說明」之後、「核心任務」之前〕

## 全系列規劃
| 篇章 | 篇章名稱 | 大綱 | 衝突強度 |
|---|---|---|---|
〔逐集列出,列數＝集數概估;大綱欄一句話,聚焦該集核心事件/轉折;衝突強度欄僅標低/中/高分級,不使用精確數字評分;整體走勢須符合所選大綱主軸走向(漸進升級→逐集遞增;懸疑揭露→維持中低至揭露點驟升;多線匯流/單元式依線判定)〕

## 故事人物設定
〔每位主要角色一小節:核心目標｜核心衝突｜貫穿弧線(標注對應篇章序號)〕

## 核心任務
依使用者主題,產出符合本文所有規範的完整文章。〔SERIES=true〕本次執行前須先對照「全系列規劃」確認本集對應篇章,產出內容不得偏離該篇章之核心事件與人物弧線轉折點。

## 輸入契約
〔編譯時依文類定義:每次執行需提供之素材(指示/前情摘要/上游文件如大綱、設定卡);多次產出型必含跨次連貫規則——前次輸出之懸置點(鉤子/伏筆/未竟事項)本次回收或明示延續〕〔SERIES=true〕另須註明:大綱版本號、人物設定版本號;本集對應「全系列規劃」之篇章序號;本集衝突強度須承接上一集標記,不得無故跳級(如上集標「低」,本集若標「高」,須於本集內文明確交代激化事件)

## 語流特徵干擾模組(隱式,生成過程中不逐字比對清單)
生成時整體維持自然語感:避免公式化過渡語與萬能連接句式,句長保持自然變化,段落開頭與結尾不落入固定句式。具體觸犯項目與量化閾值僅於草稿完成後、交稿前自檢時**一次性核對**(見下「交稿前自檢」,唯一詳列位置),生成過程中不重複展開清單,以降低詞彙級約束常駐造成的品質稅。

## 視角與細節錨定模組(內容血肉,依措辭三軸與受眾定調)
1. 通用 markdown 排版(標題 ≤3 層、粗體節制、段落留白;禁用 Obsidian 專屬語法與 frontmatter)
2. 依受眾調整深度與術語密度
3. 人性化寫作,不套模板句式:句長自然交替、以具體場景或提問開場但不規定句式、適度主觀觀點、段落留呼吸空間、結尾可呼應開頭但不重述主旨
4. 資訊密度:〔S1C 選定值〕;篇幅:〔S1C 選定值〕;風格:〔S1C 選定值〕
5. 套用風格參數與領域措辭三軸(精準度/比喻容忍度/情感濃度)
6. 台灣繁體中文

## 引用規範
- 非顯而易見的事實主張必附:[來源:作者/機構, 年份] 或 [來源:URL]
- 無明確來源的推理標註(基於現有知識推論)
- 嚴禁未標註的虛構引用或數據

## 生成前置校準(內在獨白,不輸出於最終稿,動筆前先自問)
1. 本次核心任務與目標受眾是否明確?
2. 是否維持自然語感,避免公式化過渡與萬能連接句式?(此為隱式傾向,非逐項比對;具體項目留待草稿完成後一次性核對)
3. 〔SERIES=true〕本集對應「全系列規劃」哪一篇章?人物言行是否須符合「故事人物設定」既定核心動機與弧線轉折點?本集衝突強度是否承接上一集標記、有無需要交代的激化事件?
```
