# 法律議題深度研究 + 風險評估方法論

> 從用戶「先深度研究 X 的嚴重性」需求提煉的可重複模式。

## 觸發條件

用戶提出某個法律領域/制度/行為，要求評估其「嚴重性」、「風險」、「後果」，或要求「深度研究」。

## 三步驟流程

### Step 1：議題拆解（3 個平行子題）

| 子題 | 研究視角 | 範例（預告登記） |
|:-----|:---------|:----------------|
| **A — 制度本身** | 法律依據、要件、效力、時效 | 土地法 §79-1、預告登記效力與塗銷 |
| **B — 行為人責任** | 參與者的刑事/民事/行政責任 | 代書配合設定虛偽抵押 → 刑法 §214 |
| **C — 風險鏈評估** | 組合風險、救濟路徑、嚴重性分級 | 預告登記+借款的處分限制、高風險操作 |

### Step 2：Delegation 執行

使用 `delegate_task(tasks=[])` 平行分派三個子代理，各子代理的 context 應包含：

- **研究範圍**：要查什麼、涵蓋哪些面向
- **關鍵問題**：用戶最關心的風險點
- **引用要求**：要求引用具體條號與司法見解
- **語言指示**：用中文（繁體）回報

```python
# 範例 — 分派 3 個平行研究子代理
tasks = [
    {"goal": "深度研究 A 制度：法律依據、效力、時效、實務風險", "context": "..."},
    {"goal": "深度研究 B 行為人：法律責任、常見問題、案例", "context": "..."},
    {"goal": "深度研究 C 風險鏈：組合風險、救濟路徑、嚴重性", "context": "..."},
]
```

### Step 3：結果整合

三個子代理回報後，整合為完整報告，重點擺在：

1. **法律依據表格** — 條號、效力、要件一覽
2. **風險矩陣** — 各環節的風險等級（🔴 高 / 🟡 中 / 🟢 低）
3. **比較分析** — 相似制度的效力差異（如預告登記 vs 抵押權 vs 信託）
4. **救濟路徑** — 民事/刑事/行政三管齊下
5. **結論 — 嚴重性總評**

## 三性分析框架（替代評估維度）

當用戶追問「合理性？正當性？合規性？」或要求評估某法律工具/操作的合法正當性時，使用此三維分析框架替代純嚴重性評級：

### 三性定義

| 維度 | 核心問題 | 判斷標準 |
|:-----|:---------|:---------|
| **合理性** 合理性 ❌/⚠️/✅ | 制度目的與實際用途是否匹配？ | 工具是否用於其制度設計的原意（如用預告登記擔保借款 vs 抵押權）；手段與目的是否比例相當 |
| **正當性** 正當性 ❌/⚠️/✅ | 有無法律上正當利益？ | 債權是否真實存在；有無對價；是否構成權利濫用（民法§148）；意思表示有無瑕疵 |
| **合規性** 合規性 ❌/⚠️/✅ | 具體違反哪些法條？ | 逐條比對（土法§79-1、地政士法§27、刑法§214/§339等）；代書/行為人的刑事/民事/行政責任路徑 |

### 使用時機

- 用戶以「合理性？正當性？合規性？」三個問句追問
- 需要比嚴重性評級更細緻的合法/違法邊界定性
- 分析爭議法律操作（如預告登記代借款、假債權設定、表見代理）

### 輸出結構

```markdown
## 🔴 核心結論：三性皆不成立

### 一、合理性 ❌
（制度目的不符、工具錯置）

### 二、正當性 ❌
（無請求權/無對價、權利濫用）

### 三、合規性 ❌
（逐條：土法§79-1 → 地政士法§27 → 刑法§214 → 刑法§339）

### 四、綜合評估表
| 維度 | 結果 | 說明 |
|:-----|:----:|:------|
| 合理性 | ❌ | ... |
| 正當性 | ❌ | ... |
| 合規性 | ❌ | ... |
| 代書刑事風險 | 🔴🔴🔴🔴🔴 | 刑法§214(3年以下)+除名 |
```

### 與嚴重性評估的關係

- **嚴重性評估**（Step 3 預設）：回答「這有多嚴重？」— 量級判斷
- **三性分析**（Step 3 替代）：回答「這合法嗎？」— 邊界定性
- 兩者可串聯：三性分析先行定性違法 → 嚴重性評估定量後果

## 產出格式 — 寫作原則（使用者確認 ✅ 2026-07-07）

### 中立立場
不預設違法故意。用「可能涉及」「宜注意」「相關法條框架」「由司法機關依具體事實判斷」取代「構成犯罪」「違反」「違法」「致命風險」。區分條文框架與個案事實，避免代人下結論。

### 引用格式
全文嵌入式引用：在敘述後加 `(全國法規資料庫)` `(內政部)` `(相關判決)` 等，不只在文末。法條引文可用 `law_box` 框起並附來源。

### 輸出結構（結論先行）

報告結論先行，結構化呈現：

```
## 嚴重性總評：🔴 高 / 🟡 中 / 🟢 低

### 核心發現
（2-3 句話總結論點）

### 1️⃣ 制度分析
| 項目 | 內容 |
|:-----|:-----|
| 法律依據 | 條號 + 簡要內容 | ...

### 2️⃣ 行為人責任

### 3️⃣ 風險矩陣
| 風險類型 | 情境 | 等級 | 救濟 |
|:---------|:-----|:----:|:----:|

### 結論與建議
```

## 注意事項

- 子代理為獨立 context，需在 goal/context 中傳遞完整指令
- 每個子代理應要求引用具體條號（民法 §X、土地法 §Y、刑法 §Z）
- 嚴重性評估後應給出具體可操作的救濟建議
- 不確定的部分需標示「待查」或「需律師進一步確認」
